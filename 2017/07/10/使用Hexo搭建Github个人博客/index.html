使用Hexo搭建Github个人博客

发表于 2017-07-10 | 分类于 搭建

前言

从事Android这么久，学习都是断断续续，笔记零零散散，有心情了记一下，没心情了看看就过去了，回头一看并没有留下些什么。于是建立了这个博客，一是有个固定的地方记录，二是对自己的督促。

第一篇就用此次搭建个人博客来吧。

博客是参考【参考资料】中的两篇文章创建的，此文在此基础上删除一些笔者用不上的内容，加入一些遇到的坑。

环境准备

笔者电脑为window环境。

安装Git

下载git并执行。

安装Node.js

下载Node.js并执行。

安装hexo

npm install -g hexo

搭建本地hexo博客

初始化

创建hexo文件夹（如：F:\hexo)，执行以下命令（在F:\hexo)，hexo即自动在该文件夹中生成建立博客所需的所有文件。

hexo init

安装依赖包

npm install

本地查看

至此本地hexo博客已经搭建好了，执行以下命令（在F:\hexo)，然后到浏览器输入localhost:4000查看。

hexo generate
hexo server

或者使用组合命令

hexo s -g

使用GitHub Pages建立博客

首先注册一个GitHub账号。

创建页面仓库

登录GitHub，在首页点击”New repository”。

仓库名字与账号一致，格式：GitHub账号.github.io

输入基本信息，然后点击”Create Repository”。

配置SSH key

配置SSH key让本地git项目与远程的github建立联系。

生成新的SSH Key

$ ssh-keygen -t rsa -C "邮箱地址@youremail.com"

直接3个回车即密码为空，则提交项目时不需要输入密码

在C:\Users\用户名\ .ssh下，得到两个文件id_rsa和id_rsa.pub。

添加SSH Key到GitHub

打开id_rsa.pub，复制全文。https://github.com/settings/ssh ，Add SSH key，粘贴进去。

测试

输入下面的命令，测试设置是否成功

$ ssh -T git@github.com

-v是debug选项

如果遇到如下反馈：

The authenticity of host ‘github.com (192.30.255.112)’ can’t be established.
RSA key fingerprint is SHA256:nThbg6kXUpJWGl7E1IGOCspRomTxdCARLviKw6E5SY8.
Are you sure you want to continue connecting (yes/no)?

输入yes。如果看下如下信息就说明SSH Key配置成功了。

Hi hattie0312! You’ve successfully authenticated, but GitHub does not provide shell access.

配置博客

修改全局配置文件_config.yml(参数信息见附件)。

deploy:
  type: git
  repo: #刚刚github创库地址.git
  branch: master #项目的分支，默认用主分支master。

其他参数可以根据个人喜好进行配置。

发表文章

用hexo创建文章有两种方式：

1)执行以下命令，会在\source\_posts文件夹中生成”my-first-blog.md”文件，用MarkDown编辑器打开编写即可。

hexo new "my first blog"

my first blog 为文章标题，且标题中空格自动替换成 -

2)直接在\source\_posts文件夹中新建一个md文件。

写完后，执行命令，推送到服务器上。

hexo generate
hexo deploy

或者使用组合命令

hexo d -g

建议每次执行命令前先清理缓存，每次部署前先删除 .deploy 文件夹

至此，个人博客就简单搭建完成了。

进阶

修改主题

以next主题为例。

  • 安装next到themes目录下。

    git clone https://github.com/iissnan/hexo-theme-next themes/next

  • 修改站点配置文件_config.yml。

    theme: next

    注意区分两个配置文件：

    站点配置文件：hexo 博客目录下面的 _config.yml 文件。

    主题配置文件：themes/next 目录下的 _config.yml 文件。

  • 验证主题

    启动Hexo本地站点

    hexo s --debug

    –debug 调试模式

    使用浏览器访问 http://localhost:4000，确认主题是否正确切换。

  • 根据个人喜好配置主题

图床

文章中图片可选择免费图床上传。

此文所用图片使用七牛图床，通过qrsbox工具快速上传图片。

将独立域名与GitHub Pages的空间绑定

方法一：在站点source目录下面，新建一个名为CNAME的文本文件，里面写入你要绑定的域名，比如hattie.me。

方法二：在Repository的根目录下面，新建一个名为CNAME的文本文件，里面写入你要绑定的域名，比如hattie.me。

遇到的坑

  1. 在执行hexo deploy后,出现 error deployer not found:github的错误。

    解决方案：

    npm install hexo-deployer-git --save

    删掉根目录下的文件夹：.deploy_git，重新deploy。

  2. 执行ssh-add时出现Could not open a connection to your authentication agent

    解决方案：使用Git Bash,执行以下命令开启ssh-agent

    eval "$(ssh-agent -s)"

    然后再次执行

    ssh-add -l

  3. 多个ssh key冲突

    使用github，同时还有公司的gitlab，而github对应的是QQ邮箱，gitlab对应的是公司的邮箱。不想在Github上添加公司邮箱获取的ssh key的情况下，两个ssh key冲突了。

    解决方案：

    ssh-keygen -t rsa -C "youremail@email.com" -f ~/.ssh/id_rsa_github
ssh-keygen -t rsa -C "youremail@email.com" -f ~/.ssh/id_rsa_gitlab

    1)在C:\Users\用户名\ .ssh下，分别生成id_rsa_github、id_rsa_gitlab对应github与gitlab。

    2)新建config文件(新建一个txt文本，然后将名字后缀一起改成config)，用于配置各个公私钥对应的主机

  4. ssh_exchange_identification: read: Connection reset by peer

    因为笔者密码错误次数过多，ip被服务端限制了。服务端没有权限，怎么确认呢？

    简单的方法就是切换网络，换个ip如果可以用了，那么告知服务端人员把ip解除限制吧。

  5. 发布到博客上中文乱码

    _config.yml文件中设置language: zh-CN。

附件

Hexo常用命令及简写

hexo help 
#查看帮助
hexo init 
#初始化一个目录
hexo new "postName" 
hexo n
#新建文章
hexo new page "pageName" 
#新建页面
hexo generate 
hexo g
#生成网页，可以在 public 目录查看整个网站的文件
hexo server 
hexo s
#本地预览，'Ctrl+C'关闭
hexo deploy 
hexo d
#部署.deploy目录
hexo clean 
#清除缓存

Hexo目录结构

├── .deploy #需要部署的文件

├── node_modules #Hexo插件

├── public #生成的静态网页文件

├── scaffolds #模板

├── source #博客正文和其他源文件，404、favicon、CNAME 都应该放在这里

| ├── _drafts #草稿

| └── _posts #文章

├── themes #主题

├── _config.yml #全局配置文件

└── package.json

全局配置文件_config.yml

# Hexo Configuration
## Docs: http://hexo.io/docs/configuration.html
## Source: https://github.com/hexojs/hexo/

# Site #站点信息
title:  #标题
subtitle:  #副标题
description:  #站点描述，给搜索引擎看的
author:  #作者
email:  #电子邮箱
language: zh-CN #语言

# URL #链接格式
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url:  http://yoursite.com #网址
root: /child #根目录
permalink: :year/:month/:day/:title/ #文章的链接格式
permalink_defaults:

# Directory #目录
source_dir: source #源文件目录
public_dir: public #生成的网页文件目录
tag_dir: tags #标签目录
archive_dir: archives #存档目录
category_dir: categories #分类目录
code_dir: downloads/code
i18n_dir: :lang
skip_render:

# Writing #写作
new_post_name: :title.md #新文章标题
default_layout: post #默认的模板，包括 post、page、photo、draft（文章、页面、照片、草稿）
titlecase: false #标题转换成大写
external_link: true #在新选项卡中打开连接
filename_case: 0
render_drafts: false
post_asset_folder: false
relative_link: false
highlight: #语法高亮
  enable: true #是否启用
  line_number: true #显示行号
  tab_replace:

# Category & Tag #分类和标签
default_category: uncategorized #默认分类
category_map:
tag_map:

# Date / Time format #日期时间格式
## Hexo uses Moment.js to parse and display date
## You can customize the date format as defined in
## http://momentjs.com/docs/#/displaying/format/
date_format: YYYY-MM-DD 
time_format: H:mm:ss

# Pagination #分页
per_page: 10 #每页文章数，设置成 0 禁用分页
pagination_dir: page

# Extensions #拓展插件
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: landscape #主题
exclude_generator:
plugins: #插件，例如生成 RSS 和站点地图的
- hexo-generator-feed
- hexo-generator-sitemap

# Deployment #部署
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:hattie0312/hattie0312.github.io.git
  branch: master #项目的分支，默认用主分支master。

注意：.yaml语法要求严格，配置文件的冒号“:”后面一定要有一个空格
